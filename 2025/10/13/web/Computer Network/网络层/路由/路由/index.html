路由器

路由器（Router）是一种计算机网络设备，它工作在OSI模型和TCP/IP模型中的第三层——网络层 (Network Layer)。它拥有多个输入端口和输出端口, 专门用于连接不同的网络，并负责在这些网络之间转发数据包 (Packet) 。

与工作在第二层（数据链路层）的交换机（Switch）不同：

  • 交换机：主要负责在同一个局域网（LAN）内部转发数据，它根据设备的物理地址（MAC地址）来做决策

  • 路由器：则负责连接不同的网络（例如，你家的局域网与互联网），它根据逻辑地址（IP地址）来做决策

网络层的两大功能是分组转发和路由选择, 而路由器的所有功能都围绕这两个问题展开，具体体现在以下几个方面：

逻辑寻址 (Logical Addressing): 网络层使用IP地址作为设备的逻辑地址。这个地址是分层的，包含了网络部分和主机部分，这使得路由器能够判断一个设备是否属于同一个网络。

当路由器收到一个数据包时，它会检查其“信封”上的目的IP地址。这是路由器做出所有决策的基础。它通过这个地址来识别数据包最终要去往的那个遥远的网络。

路径选择 (Path Determination / Routing)

这是路由器最“智能”的部分。路由器内部维护着一张名为路由表 (Routing Table)的动态地图。这张表详细记录了去往各个已知网络的路径信息。

一张路由表通常包含以下关键信息： - 目标网络 (Destination Network)：要去往的网络的地址。 - 子网掩码 (Subnet Mask)：用于确定目标网络的范围。 - 下一跳 (Next Hop)：要到达目标网络，应该将数据包交给的下一个路由器的IP地址。 - 出接口 (Outgoing Interface)：将数据包从本路由器的哪个物理端口（如 GigabitEthernet0/1）发出去。 - 度量值 (Metric)：一个衡量路径“好坏”的数值。度量值越小，路径通常越优。

路由器通过三种方式学习并构建这张“地图”：

  • 直连网络 (Directly Connected)：路由器启动后，会自动识别与其物理端口直接相连的网络。
  • 静态路由 (Static Routing)：由网络管理员手动配置的永久性路由。适用于网络结构简单且固定的场景。
  • 动态路由 (Dynamic Routing)：路由器之间通过运行特定的路由协议（如OSPF, BGP, RIP等）来自动交换路由信息，动态地学习网络拓扑结构，并自动计算出最佳路径。这是大型网络保持高效运行的关键。

此外, 转发表 (Forwarding Table) 是路由器用来实际执行数据包转发的表格。它通常是从路由表中提取和优化而来，结构更简单，便于快速查找。

数据包转发 (Packet Forwarding)

一个数据分组从进入路由器到离开的完整流程如下：

  1. 输入端口接收：

    • 物理层：信号从某个输入端口进入路由器，物理层将其从电信号转换为比特流 。
    • 数据链路层：数据链路层从比特流中识别出完整的帧，去掉帧头和帧尾后，将内部的数据分组（IP数据报）向上交付给网络层处理 。

  2. 网络层处理：网络层会根据分组的类型进行不同的处理：

    • 情况一：普通待转发的数据分组
      • 查表转发：网络层会提取分组首部中的目的IP地址，在 转发表 中进行快速查找 。
        • 如果找不到匹配的转发条目，该分组将被丢弃 。
        • 如果找到匹配条目，则按照条目中指示的输出端口进行转发 。
      • 首部更新：在转发前，网络层会更新数据分组首部中的某些字段，最典型的就是将 生存时间 (TTL) 减1 。
      • 交付输出：更新后的分组被向下交付给相应输出端口的数据链路层 。
    • 情况二：携带路由报文的路由分组
      • 当网络层识别出这是一个路由协议的分组时（如RIP或OSPF报文），它不会被转发，而是被送交给路由选择处理机 。
      • 路由选择处理机根据路由报文的内容来更新自己的路由表 。

  3. 交换结构 (Switching Fabric)：这是路由器的关键构件，负责将分组从输入端口物理地传送到正确的输出端口, 其速率对路由器的整体性能至关重要 。

  4. 输出端口发送：

    • 数据链路层：从网络层接收到数据分组后，将其重新封装成帧（添加新的帧头和帧尾） 。
    • 物理层：将帧看作比特流，并将其转换为相应的电信号，通过物理链路发送出去 。

此外, 路由器还具备输入缓冲区, 用于暂存新进入路由器但还来不及处理的分组, 和输出缓冲区, 用于暂存已经处理完毕但还来不及发送的分组，例如当输出链路繁忙时。

静态路由配置

静态路由配置是指由用户或网络运维人员通过路由器的相关命令，手动为路由器添加路由表条目的一种方式。这种方式不依赖于任何路由选择协议，路由信息不会自动更新或交换，完全由人工维护。

优点：这种方法实现简单，对路由器的处理开销小 。 缺点：它不能自动适应网络状态（如流量、拓扑结构等）的变化。因此通常只在小规模的网络中采用 。

直连路由 (Directly Connected Route)和非直连路由 (Non-Directly Connected Route)

当网络运维人员为路由器的各个接口配置了IP地址和地址掩码后，路由器能够自行得出它直接连接了哪些网络。这些由路由器自动生成的、指向其直连网络的路由条目，就称为直连路由 。

如图, 路由器R1的接口0配置了IP地址 192.168.1.254/24，因此R1自动知道它与网络 192.168.1.0/24 是直连的 。

同样，R1的接口1配置了IP地址 10.0.0.1/30，因此它也自动知道与网络 10.0.0.0/30 是直连的 。

这两条路由被自动记录在R1的路由表中，类型为“直连” 。

而对于那些不是直接连接的网络，路由器无法自动知道如何到达。此时，需要通过人工配置（即静态路由配置）或动态路由选择协议来告诉路由器如何到达这些网络。通过人工配置的条目就是非直连的静态路由。

在上图中, 路由器R1并不知道如何到达网络 192.168.2.0/24，因为它与该网络非直连 。

网络管理员需要手动为R1配置一条静态路由，告诉它：“要去往目的网络 192.168.2.0/24，下一跳地址是 10.0.0.2（即R2的接口地址）” 。

这条手动添加的路由条目就属于非直连路由，类型为“静态” 。

默认路由和特定主机路由

除了上述两种路由，运维人员还可以配置两种特殊的静态路由：默认路由和特定主机路由 。

默认路由 (Default Route): 当路由器在路由表中查找一个目的地址，但找不到任何匹配的路由条目时，如果没有默认路由，它会丢弃该数据包 。如果配置了默认路由，路由器就会按照默认路由条目中指示的“下一跳”地址进行转发 。

它是一种特殊的路由条目，用于匹配所有未在路由表中明确列出的目的网络地址。它相当于一个“兜底”的选项，确保路由器不会因为找不到匹配的路由而丢弃数据包 。

优点：使用默认路由可以极大地减少路由表的大小和搜索路由表所耗费的时间。

配置方式：默认路由的目的网络地址通常被配置为 0.0.0.0/0 。其中 0.0.0.0 代表任意网络，而网络前缀 /0 是最短的网络前缀，这意味着它的匹配优先级最低，只有在所有其他路由都不匹配时才会被使用 。

特定主机路由 (Specific Host Route): 这是一种专门为某一台特定的主机配置的路由条目，通常用于网络管理、测试或出于安全考虑 。

配置方式：特定主机路由的目的网络地址被配置为该主机的IP地址，并且网络前缀为 /32（地址掩码为 255.255.255.255） 。/32 是最长的网络前缀，根据“最长前缀匹配”原则，这条路由的匹配优先级是最高的。

优点：特定主机路由允许网络管理员为某个主机指定一条独特的路径，绕过常规的路由选择逻辑。这在某些情况下非常有用，例如需要对某个服务器进行特殊的流量管理或监控时。

因特网采用分层次的路由选择协议

因特网/Internet 是全球规模最大的互联网，要在如此庞大的网络中高效、准确地找到数据传输的路径，单一的路由选择策略是不可行的。因此，因特网采用了具有以下三个主要特点的路由选择协议 ：

  • 自适应 (Adaptive)：因特网采用动态路由选择，能够根据网络流量和拓扑结构的变化自动调整路由，而不是依赖人工干预 。

  • 分布式 (Distributed)：因特网中的路由器通过相互交换信息，共同协作来完成路由的计算和更新，而不是由一个中央控制器来决定所有路径 。

  • 分层次 (Hierarchical)：这是最关键的特点。它将整个庞大的因特网划分为许多较小的、可管理的部分，称为 自治系统 (Autonomous System, AS) 。

    • 一个自治系统（AS）是在一个单一技术管理机构下的路由器和网络的集合。例如，一个大型的互联网服务提供商（ISP）、一所大学或一个大型公司的网络都可以被视为一个自治系统 。
    • 目的：通过将因特网划分为多个AS，可以将复杂的路由问题分解为两个层次来处理：自治系统内部的路由选择和自治系统之间的路由选择。

两种层次的路由选择协议

基于自治系统的划分，路由选择协议也被相应地分为了两大类 ：

域内路由选择 (Intradomain Routing): 在一个自治系统内部进行的路由选择。

  • 使用的协议：内部网关协议 (Interior Gateway Protocol, IGP)。

  • 常见的IGP协议：路由信息协议 (RIP)、开放最短路径优先 (OSPF)。

  • 特点：每个自治系统可以自由选择适合自己的IGP，而无需考虑其他AS使用什么协议。例如，图4-61中，自治系统AS1内部使用RIP协议，而AS2内部使用OSPF协议，它们互不影响。

域间路由选择 (Interdomain Routing): 在不同的自治系统之间进行的路由选择。

  • 使用的协议：外部网关协议 (External Gateway Protocol, EGP)。

  • 常见的EGP协议：边界网关协议 (BGP) 是目前事实上的标准。

  • 特点：EGP负责在各个AS的边界路由器之间交换可达性信息，从而将整个因特网连接成一个整体。

在早期的RFC文档中，“路由器”被称为“网关”(Gateway)。因此，协议名称中的“网关”实际上指的就是“路由器”。所以，IGP和EGP也可以被称为内部路由协议（IRP）和外部路由协议（ERP）。

通过这种分层次的结构，因特网的路由问题被有效地简化了。AS内部的路由器只需关注内部网络的拓扑细节，而AS之间的路由器则只需关注如何到达其他AS，无需了解对方内部的复杂情况。这种“内部精细，外部概括”的策略，是因特网能够扩展到全球规模的关键所在。

路由信息协议 (Routing Information Protocol, RIP)

路由信息协议（RIP）是最早得到广泛使用的内部网关协议（IGP）之一。它是一种基于距离向量 (Distance-Vector) 算法的协议。

下面是几个RIP的关键概念：

  • 距离向量：RIP要求自治系统内的每个路由器都维护一张路由表，表中记录了从它自己到AS内其他每一个网络的“距离” 。

  • 距离度量 (Metric)：RIP使用 跳数 (Hop Count) 作为衡量到达目的网络距离的度量。路由器到其直连网络的距离定义为 1, 到非直连网络的距离定义为所经过的路由器数量加1 。

    • 且 RIP 规定一条路径最多只能包含15个路由器，即最大跳数为15 。当距离达到 16 时，即被认为是不可达。这个限制使得RIP只适用于小型互联网。

  • 路由更新：RIP路由器通过周期性地（通常每隔30秒）向其所有邻居路由器广播自己的完整路由表来交换路由信息 。此外，当网络拓扑发生变化时，路由器会立即发送更新信息，这被称为触发更新 。

  • 最佳路由选择：RIP认为“好”的路由就是“距离短”的路由，也就是所经过路由器数量最少的路径。即使某条路径带宽更高、速度更快，但只要其经过的路由器数量多，RIP也不会认为它是最佳路径。

  • 等价负载均衡：当到达同一个目的网络有多条距离（跳数）相等的路径时，RIP可以将通信流量均衡地分布到这几条等价的路径上。

RIP的三个重要特点：

  • 和谁交换信息：仅和相邻的路由器交换信息（即中间没有其他路由器的直连路由器） 。
  • 交换什么信息：交换的信息是路由器 自己的完整路由表 。
  • 何时交换信息：周期性地（例如每隔30秒）交换 。此外，当网络拓扑发生变化时，路由器会立即发送更新信息，这被称为触发更新。

RIP的基本工作过程

  1. 初始化：路由器刚开始工作时，它的路由表中只包含到达其直连网络的信息，距离为1。

  2. 信息交换与更新：每个路由器周期性地向其所有相邻路由器发送自己的路由表。收到信息的路由器会根据后述的距离向量算法来更新自己的路由表 。

  3. 收敛：经过若干次交换和更新后，网络中所有的路由器最终都会学习到到达本自治系统内所有网络的最短距离和下一跳地址。这个所有路由表都达到稳定、一致状态的过程，称为收敛。

RIP的距离向量算法

当一个路由器（如D）收到其相邻路由器（如C）发来的RIP更新报文（即C的路由表）后，它会按以下规则更新自己的路由表：

  1. 修改收到的路由信息：路由器D首先会对收到的C的路由表进行修改：将所有路由条目的下一跳地址改为C，并将所有条目的距离（跳数）加1。这是因为D要通过C才能到达那些网络，所以距离自然要增加一跳。

  2. 更新自己的路由表：然后，D将修改后的信息与自己的现有路由表进行比较，并按以下规则更新：

    • 新网络：如果收到的路由信息中包含一个D的路由表中没有的网络，则直接添加这条新路由。
    • 相同下一跳：如果到达某个目的网络的下一跳本来就是C，那么无论C通告来的新距离是变大还是变小，D都更新为这个最新的距离。
      • 这会保持路由信息的最新性，但也可能引入不稳定性（如路由环路）。
    • 不同下一跳：如果到达某个目的网络的下一跳不是C，则根据新距离与原距离的比较，采取不同的更新策略：
      • 距离更短：如果通过C到达的新距离比原来的距离更短，则D更新路由条目，将下一跳改为C，并更新距离。
      • 距离相等：如果通过C到达的距离与原来的距离相等，则D添加这条新路由，用于等价负载均衡。
      • 距离更长：如果通过C到达的新距离比原来的距离更长，则不更新，保留原来的更优路径。

RIP存在的问题

RIP最著名的问题是 “坏消息传播得慢”，也称为路由环路或距离无穷计数问题 。

问题成因：当网络中某个链路出现故障（“坏消息”）时，路由器可能不会立即收到这个故障信息，反而会从邻居那里收到一个基于旧拓扑的、错误的“好”路由信息。这会导致两个或多个路由器之间形成一个数据包的转发环路 。它们会相互更新路由，导致到达目标网络的距离不断增加，直到达到16（不可达）为止 。这个过程可能持续数分钟，严重影响网络收敛速度 。

如图, R1已经无法到达网络N1, 但是当R1收到R2发来的路由信息时，R1会被误导, 认为还可以通过R2到达网络, 因此R1会更新自己的路由表, 将到达网络的下一跳改为R2, 并将距离加1. 然而实际上, R2本来就是通过R1到达该网络的, 但是由于不知道R1那边已经断开, 错误地发送了更新报文, 而R1对于距离更短的报文是一律接受的. 现在R1又通过R2到达该网络, 这样就形成了一个环路. 数据包会在R1和R2之间不断循环, 跳数也会不断增加, 直到达到16（不可达）为止 。

解决措施： - 限制最大跳数：将最大距离限制为15，使得距离可以快速增长到16（不可达），从而打破环路 。 - 触发更新：当路由表变化时立即发送更新，而不是等待周期计时器 。 - 水平分割 (Split Horizon)：路由器不应将从一个接口学到的路由信息再通过同一个接口发回 。

注意：这些措施只能减少问题发生的概率，但无法从根本上彻底解决距离向量算法的这个固有问题 。

现在总结一下RIP的优缺点：

优点： - 实现简单，对路由器开销小 。 - “好消息”（发现更短的路径）传播得很快 。

缺点： - 最大15跳的限制使得网络规模受限 。 - 交换的是完整路由表，随着网络规模扩大，开销也随之增加 。 - “坏消息传播得慢”，导致收敛时间过长 。因此，对于规模较大的网络，应使用OSPF协议

开放最短路径优先 (Open Shortest Path First, OSPF)

开放最短路径优先 (OSPF) 协议是为了克服路由信息协议 (RIP) 的缺点而开发的一种内部网关协议 (IGP) 。它基于链路状态 (Link State) 算法，而不是RIP所使用的距离向量算法，这从根本上保证了不会产生路由环路 。OSPF具有更新效率高、收敛速度快且不限制网络规模的特点 。

OSPF的几个关键概念：

  • 链路状态 (Link State)：指本路由器与哪些路由器相邻，以及到每个相邻路由器的链路“代价(cost)”是多少 。

  • 代价 (Cost)：是一个用于衡量路径优劣的指标，可以代表费用、距离、时延或带宽等，由网络管理员决定。例如, 代价可以根据链路带宽计算（100Mb/s 除以链路带宽, 带宽越大，代价越小）

  • 邻居关系的建立：OSPF路由器之间通过周期性地交互问候 (Hello) 分组来发现和维护邻居关系 。Hello分组通常每10秒发送一次，如果40秒内未收到邻居的Hello分组，则认为该邻居不可达 。

  • 链路状态通告 (LSA)：每个路由器都会产生链路状态通告 (LSA)，其中包含了该路由器直连网络的链路状态信息以及其所有邻居路由器的链路状态信息 。

  • 链路状态更新 (LSU) 与洪泛法(Flooding)：LSA被封装在链路状态更新分组 (LSU) 中，并采用洪泛法 (Flooding) 在整个区域内发送。洪泛法指路由器将信息从所有接口发送给所有邻居，邻居再继续转发，最终使得区域内所有路由器都收到该信息 。

  • 链路状态数据库 (LSDB)：每个运行OSPF的路由器都有一个链路状态数据库 (LSDB)，用于存储收集到的所有LSA 。通过洪泛，最终区域内所有路由器的LSDB将达到一致和同步 。

  • 最短路径优先计算：当LSDB同步后，每个路由器都会基于这个完整的拓扑数据库，使用 Dijkstra提出的最短路径算法，计算出以自己为根节点到区域内所有其他节点的最短路径，从而构建出自己的路由表 。

OSPF的五种分组类型

OSPF协议定义了五种不同类型的分组来完成其工作 ：

  • 问候 (Hello) 分组：用于发现和维护邻居路由器的可达性 。
  • 数据库描述 (DBD) 分组：向邻居路由器发送自己LSDB中所有链路状态项目的摘要信息 。
  • 链路状态请求 (LSR) 分组：向邻居请求发送某些链路状态项目的详细信息 。
  • 链路状态更新 (LSU) 分组：用于洪泛发送具体的链路状态信息 。
  • 链路状态确认 (LSAck) 分组：对收到的LSU分组进行确认 。

OSPF的基本工作过程

下图是OSPF的基本工作流程：

  1. 邻居发现与建立：路由器通过发送Hello分组来发现和建立邻居关系。收到Hello分组的路由器会回复一个Hello分组，确认邻居关系的建立。
  2. 数据库同步：邻居关系建立后，路由器会通过数据库描述 (DBD) 分组交换各自的链路状态信息，确保彼此的链路状态数据库 (LSDB) 一致。
  3. 假设路由器R1收到R2的DBD分组，发现R2有一些R1没有的链路状态信息，于是R1发送链路状态请求 (LSR) 分组，向R2请求这些缺失的LSA。
  4. R2收到LSR后，会将缺失的LSA封装在链路状态更新 (LSU) 分组中，发送给R1。
  5. R1收到LSU后，更新自己的LSDB，并发送链路状态确认 (LSAck) 分组给R2，确认已收到这些LSA。
  6. 假如R1的LSDB发生了变化（例如，某个链路状态改变），它会生成新的LSA，并通过LSU分组洪泛发送给所有邻居。邻居收到后，更新自己的LSDB，并继续洪泛转发，直到整个区域内的所有路由器都收到该信息。
  7. 当LSDB同步完成后，每个路由器都会使用Dijkstra最短路径算法，基于LSDB计算出从自己到区域内所有其他节点的最短路径，并更新自己的路由表。

OSPF划分区域 (Area)和在多点接入网络中的优化

在以太网这样的多点接入网络中，如果所有路由器都互相建立邻居关系，将会产生大量的Hello分组和LSU分组，造成不必要的开销 。

为了减少分组数量，OSPF会选举一个指定路由器 (Designated Router, DR) 和一个备用的指定路由器 (Backup Designated Router, BDR)。网络中所有其他的普通路由器只与DR和BDR建立邻居关系并交换信息，而不再互相通信。这大大减少了网络中的邻居关系数量和路由更新流量。

同时, 为了使OSPF能够应用于规模非常大的网络，OSPF允许将一个自治系统（AS）再划分为若干个更小的范围，称为区域 (Area)。划分区域后，LSA的洪泛范围被限制在每个区域内部，而不是整个AS，这极大地减少了网络上的路由通信量 。

  • 主干区域 (Backbone Area)：所有区域都必须连接到一个特殊的核心区域，即主干区域（区域0或0.0.0.0）。主干区域负责连通所有其他非主干区域 。

  • 路由器角色也根据其所在区域的不同而有所区别：

    • 区域内路由器：所有接口都在同一个区域内的路由器, 只负责该区域内的路由选择和转发 。
    • 主干路由器：至少有一个接口连接到主干区域的路由器 。它们负责在主干区域和其他非主干区域之间传递路由信息 。
    • 区域边界路由器 (ABR)：连接主干区域和一个或多个非主干区域的路由器。它们负责在不同区域之间传递路由信息 。
    • 自治系统边界路由器 (ASBR)：负责与本AS之外的其他AS交换路由信息的路由器 。

边界网关协议 (Border Gateway Protocol, BGP)

边界网关协议 (BGP) 属于外部网关协议 (EGP) 类别，专门用于在不同的 自治系统 (AS) 之间进行路由选择。

首先, 我们需要明确一个概念: BGP 寻找的并非“最佳路由”. 在不同的自治系统之间，寻找一条统一标准下的“最佳”路由是无意义的，主要原因如下：

  • 度量标准不统一：不同的自治系统可能使用完全不同的标准来衡量路径的好坏。例如，AS1可能以“时延”为度量，AS2以“跳数”为度量，而AS3则以“带宽”为度量 。由于没有统一的度量单位，因此无法在多条跨越不同AS的路径之间进行优劣比较。

  • 策略驱动路由：AS之间的路由选择必须考虑很多路由协议本身之外的策略因素 。这些策略通常由网络管理员根据政治、经济、安全等需求进行设置 。

    • 安全策略：例如，一个国家内部的两个站点通信时，应避免数据报绕道经过某些有安全威胁的国家 。
    • 经济策略：例如，AS3可能不愿意让不相关的AS4和AS5的数据报免费穿过自己的网络，而AS2则可能提供付费的 transit (中转) 服务 。

因此，BGP协议的设计目标是力求寻找一条能够到达目的网络且比较好的、不会兜圈子的路由，而不是寻找一条绝对的“最佳”路由。

BGP 发言人 (BGP Speaker)

在配置BGP时，每个AS的管理员需要选择至少一个路由器作为该AS的“BGP发言人” 。这个发言人通常就是位于AS边界的BGP边界路由器。不同AS的BGP发言人之间通过以下步骤交换路由信息 ：

  • 建立TCP连接：首先在两个发言人之间建立一个TCP连接，使用的端口号为179。
    • 使用TCP连接交换路由信息的两个BGP发言人，彼此称为对方的邻站 (neighbor) 或 对等站 (peer) 。
  • 建立BGP会话：在此TCP连接之上交换BGP报文，以建立BGP会话 。
  • 交换路由信息：利用BGP会话来交换路由信息，例如通告新路由、撤销过时路由等。
    • BGP发言人除了运行BGP协议外，还必须运行其所在AS内部使用的IGP协议（如RIP或OSPF）。

我们要知道, BGP发言人交换的是“网络可达性信息”，即要到达某个目标网络所需要经过的一系列自治系统。这个AS序列被称为路径向量。当BGP发言人互相交换了路径向量信息后，每个发言人就可以根据这些信息来构建自己的路由表, 即构造出到达各个目的网络的路径。

此外, 路径向量机制可以非常有效地避免路由环路。当一个BGP发言人收到一条路由信息时，它会检查其路径向量。如果发现路径向量中已经包含了自身的AS号，那么就说明采用这条路径会形成环路，因此会丢弃这条路由信息。

BGP-4 的四种报文

BGP-4是目前使用最广泛的版本。它规定了四种主要的报文类型 ：

  • OPEN (打开) 报文：用于与相邻的另一个BGP发言人建立对等关系，进行通信初始化 。
  • UPDATE (更新) 报文：这是BGP的核心报文，用于通告新的路由信息，或撤销多条过时的路由 。一个UPDATE报文可以撤销多条路由，但一次只能增加一条新路由 。
  • KEEPALIVE (保活) 报文：在BGP会话建立后，双方会周期性地（通常每30秒）发送KEEPALIVE报文，以确认邻站的连通性，维持会话状态 。
  • NOTIFICATION (通知) 报文：用于在检测到差错时，向对等站发送通知 。

BGP在刚运行时会交换整个BGP路由表，但在之后仅在路由发生变化时才发送UPDATE报文，这样做可以节省网络带宽和路由器的处理开销 。
